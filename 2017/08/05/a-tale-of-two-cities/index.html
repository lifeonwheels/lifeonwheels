A Tale of Two Cites

2017-08-05 PV:

Windows and Mac are two dominant desktop systems for personal usage in any enterprise. My current company provides Windows laptop as offical employee device. Using Windows laptop bring lots of pain to me and other developers since Windows is not developer-friendly. Fortunately bosses decided provide MacBook to us for daily work, and I am the first pilot. I have been using MacBook for one week and never touch my Windows laptop again. It works very well so far. Below are some notes/tips for using MacBook.

Network Access

This can be done be some simple scripts. It use apple-buildin command, networksetup, to control proxies behavior. Using “sudo” without password

Many tasks like starting/stopping services, installing packages, and invoking on.sh and off.sh I mentioned in last section, sudo and password are required. To avoid typing password, add this line to /etc/sudoers file:

kaihan ALL=(ALL) NOPASSWD:ALL (here “haihan” is my login account)

Premission

During accessing some system directories like /usr, /usr/lib, I encountered “premission denied” issue since these folders are assign to “root” user and “wheel” group. However my account is belong to “admin” group. “wheel” group was a historical legacy of BSD UNIX (and unfortunately MacOS was port from BSD UNIX) and it was very similar with “admin” group. To grant the premission of wheel group, just simply add my account (kaihan) to “wheel” group,

——- /etc/group ——-
wheel::0:root -> wheel::0:root:kaihan (here “haihan” is my login account)

Access SharePoint

SharePoint sites are accessable from Mac. However, if you use Chrome, you are prompted to type crendential to visit SharePoint site every time. If you use Firefox or Safari, no crendential needed and you can auto login as same as in Windows.

The reason of the different behavior is Chrome has more sensitive security control. If there is solution found, I will update the content.
